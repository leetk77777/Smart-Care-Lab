# 데모 시나리오

## 목표

단순 CRUD가 아니라 사회문제 해결형 시스템임을 보여주기 위해 상태 변화가 명확한 시연 흐름을 제공합니다.

## 시나리오 1: 정상

- 아침 이후 움직임이 주기적으로 감지됨
- 문 열림 없음
- 실내온도 24도
- 위험 점수 0~39점

## 시나리오 2: 주의

- 오전 9시 이후 활동이 부족함
- 평소 활동 시간과 다른 패턴 발생
- 위험 점수 40~69점
- 보호자 확인 권고 알림 생성

## 시나리오 3: 위험

- 3시간 이상 움직임 없음
- 실내온도 32도 이상
- 새벽 문 열림 감지
- 위험 점수 70점 이상
- 보호자 긴급 확인 알림 생성

## 발표 팁

1. 개인정보 보호를 먼저 강조합니다.
2. 대시보드에서 위험 점수와 판단 사유를 보여줍니다.
3. 데모 버튼으로 상태가 변화하는 장면을 보여줍니다.
4. 실제 센서가 없어도 심사가 가능한 오픈소스 구조임을 설명합니다.
