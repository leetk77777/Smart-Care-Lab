# 발표용 프로젝트 설명

## 1. 문제 정의

독거노인의 응급상황과 고독사는 발견 지연이 가장 큰 위험요소입니다. 기존의 카메라 기반 모니터링은 사생활 침해 우려가 크고, 실제 가정에 도입하기 어렵습니다.

## 2. 해결 아이디어

이 프로젝트는 카메라 없이 생활패턴 신호만 수집합니다. 움직임, 문 열림, 온습도, 조도 데이터를 분석하여 정상, 주의, 위험 상태를 판단합니다.

## 3. 차별점

- 개인정보 침해를 줄인 비영상 모니터링
- 실제 센서 없이도 가상 센서 시뮬레이터로 즉시 시연 가능
- Rule-based 분석에서 머신러닝 이상탐지로 교체 가능한 구조
- 지자체, 복지관, 보호자 앱으로 확장 가능한 REST API 설계

## 4. 시스템 구성

- React: 심사위원이 직관적으로 볼 수 있는 대시보드
- Spring Boot: 사용자, 센서 이벤트, 알림 이력 관리
- FastAPI: 위험도 분석 모듈
- PostgreSQL: 생활패턴 및 알림 이력 저장
- Simulator: 정상 → 주의 → 위험 시나리오 생성

## 5. 데모 흐름

1. 정상 데모: 움직임과 적정 온습도가 감지되어 정상 상태 표시
2. 주의 데모: 오전 9시 이후 활동 부족과 생활패턴 이탈로 주의 상태 표시
3. 위험 데모: 장시간 움직임 없음, 위험 온도, 새벽 문 열림으로 위험 상태 표시

## 6. 기대 효과

복지 담당자는 위험도가 높은 가구부터 확인할 수 있고, 보호자는 사생활 침해 없이 안부 상태를 확인할 수 있습니다. 실제 센서를 연결하면 저비용 돌봄 안전망으로 확장할 수 있습니다.

## 7. 향후 확장

- ESP32 + PIR + 문열림 + 온습도 센서 연동
- MQTT 기반 실시간 수집
- 보호자 앱 푸시 알림
- 지자체 관제 대시보드
- 머신러닝 기반 개인별 이상탐지
